The software stack

STM32 development on the F746G-DISCO board revolves around three main tools — STM32CubeMX, STM32CubeIDE, and TouchGFX — plus a handful of supporting pieces that sit underneath them; together they take you from a blank chip configuration all the way to a running, graphical application on the board.

STM32CubeMX (v6.17.0)

CubeMX is a graphical configuration tool. Instead of hand-writing every hardware initialisation routine, you use its point-and-click interface to decide which pins do what (GPIO output, UART, SPI, …), how fast the processor clock runs, and which built-in peripherals (timers, ADC, USB, …) are turned on. When you are happy with the setup you click Generate Code and CubeMX writes all the C startup and initialisation code for you automatically.

All of your choices are stored in a single project file with the extension .ioc. Think of the .ioc file as the "source of truth" for your hardware configuration — you re-open it in CubeMX any time you need to change a pin or enable a new peripheral.

STM32CubeIDE (v2.1.0)

CubeIDE is the integrated development environment (IDE) — the program where you spend most of your time. Inside CubeIDE you:

  • Write and edit your C/C++ application code.
  • Compile (also called build) it into a binary that the microcontroller can execute.
  • Flash that binary onto the board — copy it into the chip's internal flash memory so it survives a power cycle.
  • Debug it step-by-step: pause execution, inspect variables, set breakpoints.

CubeIDE ships with the CubeMX engine built in, so you can open and edit the .ioc file and regenerate code without ever leaving the IDE.

TouchGFX (v4.26.1)

TouchGFX is a drag-and-drop designer for building the graphical user interface (GUI) — the screens and interactive widgets that appear on the board's 4.3-inch LCD. You arrange buttons, images, labels, and animations visually inside the TouchGFX Designer application; it then generates the C++ code that drives those visuals at runtime.

Under the hood TouchGFX talks directly to the display hardware: the LTDC (LCD-TFT Display Controller) that pushes pixels to the screen, and the Chrom-ART / DMA2D accelerator that blends and copies graphics in hardware so the CPU stays free for your application logic.

Supporting cast

HAL — Hardware Abstraction Layer
ST's ready-made C library that ships with every generated project. Instead of poking hardware registers directly (which requires reading hundreds of pages of the chip's reference manual), you call simple functions like HAL_GPIO_TogglePin() or HAL_UART_Transmit(). HAL handles the register details for you.
ST-LINK
A small debugger/programmer chip that is physically soldered onto the DISCO board. A single USB cable between your PC and the board's ST-LINK USB connector gives you three things at once: flashing new firmware, step-by-step debugging, and a virtual serial (COM) port for printf-style logging.
The .ioc file
Your project's hardware configuration, saved in XML format by CubeMX. Check it into version control alongside your source code — it is the recipe that lets CubeMX regenerate the correct initialisation code after any configuration change.
Note ST has introduced a newer generation toolchain — STM32CubeMX2 and HAL2 — with a redesigned API. As a beginner, stay on the classic CubeMX 6.x + HAL v1 stack. The vast majority of tutorials, ST application notes, and community examples still target this generation, so you will find far more help online.

How they fit together

Here is the flow in plain English:

  1. Configure — open your .ioc file in CubeMX, set up pins and peripherals, then click Generate Code.
  2. Generate — CubeMX writes the C startup files and HAL initialisation code into your CubeIDE project folder.
  3. Code & Build — write your application logic in CubeIDE, then build to produce a .elf / .bin binary.
  4. Flash — CubeIDE sends the binary over USB through the on-board ST-LINK programmer into the chip's flash memory.
  5. Run — the STM32F746G executes your code; reset the board and it starts automatically.

TouchGFX feeds into step 2: its Designer generates C++ screen and widget code that lands directly inside the same CubeIDE project, so the build step compiles everything together.

Tip For beginners: install STM32CubeIDE (it already includes CubeMX) and TouchGFX Designer, then spend most of your time inside CubeIDE. You only need to open the standalone CubeMX or TouchGFX Designer apps when CubeIDE's built-in versions lag behind a required feature.
