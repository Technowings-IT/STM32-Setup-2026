Tutorial 2 — Button input & UART printf

Goal: read the user button and print a message to your PC over the ST-LINK serial port.

What you'll learn

  • Reading a GPIO input
  • Enabling a UART
  • Redirecting printf to the UART
  • Viewing serial output on the PC

Prerequisites

  • Finished Tutorial 1
  • You know your ST-LINK COM port number from Install & setup

Steps

Step 1 — Create the project

Create a new project named button_uart from the board selector (same as Tutorial 1), or extend your blink project.

Step 2 — Button pin

In the .ioc file, set pin PI11 to GPIO_Input.

Note On this board B1 reads high when pressed (it has an external pull-down resistor).

Step 3 — Enable the UART

In the .ioc file, enable USART1 and set Mode to Asynchronous. Confirm the pins are PA9 (TX) and PB7 (RX) — these route to the ST-LINK virtual COM port. Set speed to 115200 baud, 8 data bits, no parity, 1 stop bit (8N1).

Step 4 — Generate code

Press Ctrl+S and click Yes to generate code. A handle named huart1 now exists in the generated files.

Step 5 — Redirect printf to the UART

Open Core/Src/main.c. Add the stdio.h include inside the includes user-region:

/* USER CODE BEGIN Includes */
#include <stdio.h>
/* USER CODE END Includes */

Then add the __io_putchar function inside the functions user-region (Section 4):

/* USER CODE BEGIN 4 */
int __io_putchar(int ch)
{
  HAL_UART_Transmit(&huart1, (uint8_t *)&ch, 1, HAL_MAX_DELAY);
  return ch;
}
/* USER CODE END 4 */
Tip Always place your code inside the USER CODE BEGIN / USER CODE END comment blocks. Code outside these regions is deleted when you regenerate from the .ioc.

Step 6 — Print on press

In main.c, find the main loop and add the button-reading logic:

/* USER CODE BEGIN WHILE */
while (1)
{
  if (HAL_GPIO_ReadPin(GPIOI, GPIO_PIN_11) == GPIO_PIN_SET)
  {
    printf("Button pressed!\r\n");
    HAL_Delay(250);   /* simple debounce */
  }
  /* USER CODE END WHILE */

  /* USER CODE BEGIN 3 */
}
/* USER CODE END 3 */
Note HAL_Delay(250) waits 250 ms after each press. This prevents one physical press from flooding the terminal with dozens of messages (debouncing).

Step 7 — Build and flash

Click the hammer icon to build. Fix any errors, then click the green Run (play) button to flash the board.

Step 8 — Open a serial terminal on the PC

Use Tera Term or PuTTY:

  • Choose Serial connection type
  • Pick your ST-LINK COM port (the one identified during Install & setup)
  • Set speed to 115200, data bits 8, parity None, stop bits 1
  • Click Connect / Open
✅ Checkpoint Each press of B1 prints Button pressed! in the terminal.

What just happened

  • GPIO input reads a pin's voltage as high or low. HAL_GPIO_ReadPin returns GPIO_PIN_SET (1) when the voltage is high and GPIO_PIN_RESET (0) when it is low.
  • UART sends bytes one after another over a single wire. The ST-LINK chip on the board converts this to USB so your PC sees it as a COM port.
  • Baud rate is bits per second and must match on both ends — if the sender says 115200 and the receiver expects something different, you see garbage.
  • __io_putchar is the low-level hook the C standard library's printf calls for every character it outputs. By redirecting it to HAL_UART_Transmit, every printf call automatically goes out over USART1 to your terminal.
Watch out If you see garbage characters, the baud rates don't match — set both ends to 115200. Always end lines with \r\n.
