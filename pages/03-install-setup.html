Install & setup (Windows)

This page walks you through every tool you need before writing your first line of STM32 code. All downloads come from st.com, are free, and — since CubeMX 6.16 — no ST account is required.

What to install

All three are free downloads from st.com, and no ST account is required (account-free since CubeMX 6.16). Since CubeIDE 2.0 they are separate apps — install all three:

  1. STM32CubeIDE 2.1.0 — editor, compiler, flasher, debugger. Accept the ST-LINK USB driver prompt during install.
  2. STM32CubeMX 6.17.0 — standalone pin/clock/peripheral configurator. Required separately — it is no longer inside CubeIDE.
  3. TouchGFX Designer 4.26.1 — the UI designer and project scaffolder you start from for this board.
Note If you used older STM32 tutorials that say "CubeIDE includes CubeMX", that changed in CubeIDE 2.0 — you now install CubeMX yourself.

Install order & steps

  1. From st.com, download and run the STM32CubeIDE 2.1.0 Windows installer. Accept the license and the ST-LINK USB driver when prompted.
  2. Download and run STM32CubeMX 6.17.0 (standalone). Follow the prompts.
  3. Download and run TouchGFX Designer 4.26.1. Follow the prompts.

First launch

Open STM32CubeIDE from the Start menu. On first launch it asks you to choose a workspace folder — this is simply the folder where your projects will be stored. You can accept the suggested path or pick your own. Click Launch. You will land on the Information Center welcome screen.

Connect the board

Plug a USB data cable from your PC into the ST-LINK USB connector (CN14) on the board. A power LED lights up, Windows installs the ST-LINK driver if needed, and a virtual COM port appears.

Watch out Use a USB data cable, not a charge-only cable. Charge-only cables carry power but no data, so the board will appear to power on but your PC will not detect it.

Verify the connection

To confirm everything is recognised by Windows:

  1. Right-click the Start button and choose Device Manager.
  2. Expand Ports (COM & LPT).
  3. You should see an entry such as STMicroelectronics STLink Virtual COM Port (COM3). The exact number (COM3, COM4, …) varies per machine — make a note of it, you will need it in Demo 3 (UART).
  4. Optionally, if STM32CubeIDE later prompts you to upgrade the ST-LINK firmware, accept the upgrade (you can also do this at any time through STM32CubeProgrammer).
✅ Checkpoint Tools installed, board powered, and the ST-LINK together with its virtual COM port appear in Device Manager.
Watch out If no COM port appears or you see "ST-LINK not found", see Troubleshooting.
