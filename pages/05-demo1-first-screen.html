Demo 1 — Your first screen

Goal: Display text (and optionally an image) on the board's 4.3-inch, 480×272 LCD using TouchGFX 4.26.1.

What you'll learn

  • Starting a TouchGFX project from an application template
  • Placing widgets on a screen in TouchGFX Designer
  • Generating C++ code from the Designer
  • Building and flashing a GUI application from STM32CubeIDE

Prerequisites

  • STM32CubeIDE 2.1.0 installed and working
  • TouchGFX 4.26.1 installed (standalone app)
  • Board verified and connected — see the Install & verify page if you haven't done this yet

Steps

🎨 TouchGFX Step 1 — New project from the board template

Open TouchGFX Designer 4.26.1 → Create New. Choose the STM32F746G-DISCO application template, name it demo1_firstscreen, click Create.

Note The template pre-configures LTDC, DMA2D, the SDRAM framebuffer, and QSPI — that's why graphics "just work" on this board.

🎨 TouchGFX Step 2 — Add a TextArea

The canvas is the screen at 480×272. From the widget panel, drag a TextArea onto it. With it selected, set the text to Hello, STM32! and pick a font/size/colour. Optional: drag an Image widget and choose a bundled asset.

🎨 TouchGFX Step 3 — Generate code

Click Generate Code. TouchGFX writes the screen C++ into TouchGFX/generated/ plus the CubeIDE project files.

⌨️ CubeIDE Step 4 — Import, build, flash

In STM32CubeIDE: File ▸ Open Projects from File System… → Directory… → select the demo1_firstscreen folder → import. Click the hammer to build, then the green play to flash over ST-LINK.

If new files don't show Select the project and press F5 (Refresh).
✅ Checkpoint Hello, STM32! appears on the LCD. If you added an image widget, it should be visible too.

What just happened

TouchGFX draws your screen into a framebuffer — a large block of memory that holds the colour value of every single pixel on the display. On the STM32F746G-DISCO, this framebuffer lives in the external SDRAM (the on-board chip provides enough RAM for a full 480×272 frame at 16-bit colour). The LTDC (LCD-TFT Display Controller) peripheral reads that framebuffer continuously and drives the physical LCD panel, so whatever is in memory appears on screen.

Whenever TouchGFX needs to redraw a widget, it uses Chrom-ART (DMA2D) — a hardware accelerator built into the STM32F7 — to fill and blend pixels much faster than the CPU could on its own.

The Designer is the visual editor where you arrange widgets. Generate Code translates your layout into C++ classes (one per screen) that call the TouchGFX rendering API at runtime. You write your application logic by overriding callbacks in those generated classes.

Open the project tree and find TouchGFX/gui/ (your code) and TouchGFX/generated/ (never edit). See Workflow & project structure for the full map.

Alternative You can instead start in standalone CubeMX, enable the TouchGFX middleware, configure LTDC/DMA2D and the external memories yourself, set the toolchain to STM32CubeIDE, then open the Designer. The board template just does all that for you — recommended for beginners.
