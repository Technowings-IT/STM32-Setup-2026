Troubleshooting & glossary

Something not working? Find your symptom in the table below, then look up any unfamiliar terms in the glossary.

Common problems

Symptom Fix / cause
ST-LINK not found / no COM port appears in Device Manager Reinstall the ST-LINK USB driver from ST's website. Try a different USB port and make sure you are using a data cable, not a charge-only cable. If STM32CubeIDE prompts you to upgrade the ST-LINK firmware, accept — or use STM32CubeProgrammer to do the upgrade manually.
"No STM32 target found" or "Error in initializing ST-LINK" Check that the board is powered (the LCD backlight or an LED should be on). Confirm the debug interface is set to SWD in your project's debug configuration. Make sure no other program is holding the debugger open — close STM32CubeProgrammer or any other IDE instance that may have connected first.
Flash or erase fails Open STM32CubeProgrammer and run a full chip erase. Check that the BOOT0 pin/jumper is in the normal run position (not bootloader mode). If problems persist, try lowering the SWD clock speed in your IDE's debug settings.
TouchGFX files were not generated You must click Generate inside the TouchGFX module or TouchGFX Designer — it does not happen automatically. Also make sure that in STM32CubeMX the Toolchain / IDE is set to STM32CubeIDE before you click Generate Code.
LCD stays completely blank after flashing a TouchGFX app The SDRAM and LTDC clocks are most likely not configured. Use the official STM32F746G-DISCO application template as your starting point and confirm you selected the correct board template — it pre-configures the display hardware for you.
Serial terminal shows garbage characters Baud rate mismatch. Set both your firmware (the MX_USART1_UART_Init call) and your terminal program to 115200 baud, 8N1 (8 data bits, no parity, 1 stop bit).
Code you wrote has disappeared after regenerating from CubeMX CubeMX overwrites anything outside the protected regions. Always place your custom code inside the /* USER CODE BEGIN … */ and /* USER CODE END … */ markers — CubeMX preserves everything between them.

Glossary

Term Meaning
MCU Microcontroller Unit. A tiny computer on a single chip — it has a processor, memory, and input/output pins all built in. The STM32F746NG is the MCU on your DISCO board.
HAL Hardware Abstraction Layer. A library provided by ST that wraps the low-level register details into easy-to-call C functions like HAL_GPIO_WritePin(). It lets you drive hardware without reading every register datasheet first.
GPIO General-Purpose Input/Output. A pin on the MCU that you can freely control in software — set it HIGH or LOW (output) or read whether something is connected to it (input). Used for LEDs, buttons, sensors, etc.
UART Universal Asynchronous Receiver/Transmitter. A simple serial communication protocol that sends data one bit at a time over two wires (TX and RX). Commonly used to print debug messages to a PC terminal.
Baud The number of signal changes per second on a serial line — in practice, roughly equal to bits per second. Both ends of a UART link must agree on the same baud rate (e.g., 115200) or the data looks like garbage.
Flash vs RAM Flash is non-volatile storage — it keeps your program code even when power is off (like a hard drive). RAM is fast but volatile — it holds variables while running, but is wiped on reset. The STM32F746NG has 1 MB Flash and 340 KB RAM on-chip.
Framebuffer A region of RAM that holds the pixel colour data for the entire screen. The display controller reads this memory continuously to refresh what you see on the LCD.
LTDC LCD-TFT Display Controller. The hardware block inside the STM32F7 that reads the framebuffer and drives the physical LCD panel with the correct timing signals. You configure it once (usually via CubeMX) and it runs automatically.
DMA2D / Chrom-ART A dedicated hardware accelerator for 2-D graphics operations — copying rectangles, filling colours, and blending layers — without using the main CPU. TouchGFX uses it automatically to keep animations smooth. ST markets it under the name Chrom-ART Accelerator.
IDE Integrated Development Environment. A program that combines a code editor, compiler, and debugger in one window. For STM32 development you use STM32CubeIDE.
Toolchain The set of programs that turn your C source code into a binary the MCU can run — mainly the compiler (GCC for ARM), linker, and related utilities. STM32CubeIDE bundles a toolchain for you.
.ioc file The project configuration file created and read by STM32CubeMX. It stores all your pin assignments, clock settings, and middleware choices. Opening it lets you change the hardware configuration and regenerate the C initialisation code.
MVP Model-View-Presenter. The software design pattern used by TouchGFX. The Model holds application data, the View describes the screen layout, and the Presenter connects them. Keeping these three parts separate makes UI code easier to maintain.
ST-LINK The small debug chip built onto the DISCO board (or a separate USB dongle). It bridges your PC to the MCU's debug interface over USB, letting you flash new firmware and step through code line by line in the debugger.
VCP Virtual COM Port. The ST-LINK also exposes a serial (UART) connection to your PC as a fake COM port — no extra USB-to-serial adapter needed. You can use it with any terminal program (PuTTY, Tera Term, etc.) to receive debug output from your firmware.
