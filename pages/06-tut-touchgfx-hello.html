Tutorial 3 — TouchGFX “Hello, STM32!”

Goal: Display text (and optionally an image) on the board's 4.3-inch, 480×272 LCD using TouchGFX 4.26.1.

What you'll learn

  • Starting a TouchGFX project from an application template
  • Placing widgets on a screen in TouchGFX Designer
  • Generating C++ code from the Designer
  • Building and flashing a GUI application from STM32CubeIDE

Prerequisites

  • STM32CubeIDE 2.1.0 installed and working
  • TouchGFX 4.26.1 installed (the Designer opens automatically from within CubeIDE or as a standalone tool)
  • Board verified and connected — see the Install & Setup page if you haven't done this yet

Steps

Step 1 — Create a new project from the application template

Open TouchGFX Designer 4.26.1, then click Create New. In the template browser, choose the STM32F746G-DISCO application template. Name the project hello_gfx and click Create.

Note The application template auto-configures LTDC, DMA2D, the SDRAM framebuffer, and QSPI — that's why it "just works" on this board. You don't need to set up any of the graphics hardware yourself.

Step 2 — Add a Text widget to the canvas

The canvas shows the board's screen at its native resolution: 480×272 pixels. In the widget panel on the left, find the Text (or TextArea) widget and drag it onto the canvas wherever you'd like the text to appear.

Step 3 — Set the text content and style

With the widget selected, look at the right-hand properties panel. Set the text value to Hello, STM32!. Adjust the font, size, and color to your liking.

Optional: drag an Image widget from the widget panel onto the canvas and select one of the bundled images from the asset picker.

Step 4 — Generate code

Click Generate Code in the top-right corner of the Designer. TouchGFX will write all the C++ source files for your screens and widgets, plus the STM32CubeIDE project files, into the hello_gfx folder.

Tip You must regenerate code every time you make changes in the Designer. Think of "Generate Code" as saving your design and compiling it into the project.

Step 5 — Build and flash from STM32CubeIDE

Switch to STM32CubeIDE. Go to File ▸ Open Projects from File System…, click Directory…, and select the hello_gfx folder. Import the project.

Click the hammer icon to build. When the build finishes with no errors, click the green play button to flash the firmware to the board.

✅ Checkpoint Hello, STM32! appears on the LCD. If you added an image widget, it should be visible too.

What just happened

TouchGFX draws your screen into a framebuffer — a large block of memory that holds the color value of every single pixel on the display. On the STM32F746G-DISCO, this framebuffer lives in the external SDRAM (the on-board chip provides enough RAM for a full 480×272 frame at 16-bit color). The LTDC (LCD-TFT Display Controller) peripheral reads that framebuffer continuously and drives the physical LCD panel, so whatever is in memory appears on screen.

Whenever TouchGFX needs to redraw a widget, it uses Chrom-ART (DMA2D) — a hardware accelerator built into the STM32F7 — to fill and blend pixels much faster than the CPU could on its own.

The Designer is the visual editor where you arrange widgets. Generate Code translates your layout into C++ classes (one per screen) that call the TouchGFX rendering API at runtime. You write your application logic by overriding callbacks in those generated classes.

The CubeMX route (alternative)

If you prefer to start from scratch or need to combine TouchGFX with a custom peripheral configuration, you can open STM32CubeMX, select the STM32F746G-DISCO board, and enable the TouchGFX graphics middleware in the middleware section. You then configure LTDC, DMA2D, FMC (for SDRAM), and QSPI manually, set the toolchain to STM32CubeIDE, and generate the project. CubeMX will include the TouchGFX module, and from there you open the Designer to add screens and widgets.

Note Instead of a template you can start in STM32CubeMX, enable the TouchGFX graphics middleware, configure LTDC/DMA2D and the external memories yourself, set the toolchain to STM32CubeIDE, then generate from the TouchGFX module. The application template just does all of this for you.
Watch out If no TouchGFX files appear after generating, either you didn't generate from the TouchGFX module, or the toolchain wasn't set to STM32CubeIDE before generating. See Troubleshooting.
