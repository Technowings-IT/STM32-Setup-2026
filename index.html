STM32F746G-DISCO — Getting Started

A hands-on, fully offline guide that takes someone new to embedded from zero to working projects on the STM32F746G-DISCO. It opens by double-clicking this file — no internet needed.

The big idea For a board with a display, TouchGFX is the entry point: you design the UI and it scaffolds the entire project — the CubeMX .ioc and the CubeIDE project. From there you round-trip: tweak the UI in TouchGFX, enable any pin/peripheral in standalone CubeMX, and write your behaviour (the "backend") in CubeIDE. This guide is built around that loop.

The software stack

ToolVersionWhat it's for
TouchGFX Designer4.26.1Draw screens/widgets; generates the whole project (incl. the CubeMX .ioc and CubeIDE files) — your entry point for this board
STM32CubeMX6.17.0Standalone app. Edit the .ioc: enable pins, peripherals, clocks; regenerate init code
STM32CubeIDE2.1.0Write, build, flash, and debug your C/C++ backend (no longer bundles CubeMX)
HALHAL1 (classic)ST's ready-made C functions the generated code calls
Note A newer CubeMX2 / HAL2 line exists, but it is less mature — beginners should stay on classic CubeMX 6.x + HAL1, which this guide uses.

  1. Read Introduction (Software stack, Board overview) and do Setup (Install & verify).
  2. Read Workflow — it explains the TouchGFX-first loop and the project folder layout the demos rely on.
  3. Do the demos in order: D1 First screen → D2 Interactive UI → D3 Screen ↔ hardware. Each builds on the last.
